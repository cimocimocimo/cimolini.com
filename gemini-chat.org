*** Method 1: Using the Command Line (Quickest)
If you have Pandoc installed, run this on the buffer:
1.  Select the text (or =C-x h= for the whole buffer).
2.  Press =M-|= (=shell-command-on-region=).
3.  Type: =pandoc -f markdown -t org=
4.  The converted text will appear in a new buffer called =*Shell Command Output*=.

*** Method 2: Replace Buffer Content
To replace the current Markdown buffer with Org-mode text directly:
1.  =C-u M-|=

In Org-mode, tables are aligned automatically.

*** 1. The Trigger
While your cursor is anywhere inside the table, press:
- *=TAB=*: Realigns the table and moves to the next cell.
- *=S-TAB=*: Realigns and moves to the previous cell.
- *=C-c C-c=*: Realigns the table without moving the cursor.

*** 2. Creating a Table from Scratch
1. Type =| Name | Email |= and press *=TAB=*. Org will create the borders.
2. To add a horizontal separator line: Type =|-= and press *=TAB=*. Org will expand it into =|-------+-------|=.

*** 3. Converting Existing Text to a Table
If you have CSV or tab-separated text:
1. Highlight the region.
2. Press *=C-c |=* (=org-table-create-or-convert-from-region=).
3. It will ask for the separator (comma, tab, etc.) and format it instantly.

*** 4. Useful Column Commands
- *=M-Left/Right=*: Move a column left or right.
- *=M-S-Left=*: Delete the current column.
- *=M-S-Right=*: Insert a new column.

#+begin_reasoning
**Defining Sitemap Standards**

I've determined the user is seeking the right sitemap format. The focus is now on confirming that the standard format is the XML Sitemap protocol (`sitemap.xml`). I've also identified a plain text format as an alternative for a simpler implementation, depending on their needs.
        #+end_reasoning

The standard and most SEO-friendly format is *XML*.

*/ 1. Standard XML Sitemap (=sitemap.xml=)
The file must be UTF-8 encoded and contain the =<urlset>= namespace.

#+begin_src xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://yourdomain.com/</loc>
    <lastmod>2023-10-27</lastmod>
    <changefreq>monthly</changefreq>
    <priority>1.0</priority>
  </url>
  <url>
    <loc>https://yourdomain.com/blog/my-post</loc>
    <lastmod>2023-10-25</lastmod>
    <priority>0.8</priority>
  </url>
</urlset>
#+end_src

*/ 2. Key Tags
- =<loc>=: *Required.* The full absolute URL of the page.
- =<lastmod>=: Optional. Date of last modification (YYYY-MM-DD).
- =<changefreq>=: Optional. How often the page likely changes (always, hourly, daily, weekly, monthly, yearly, never).
- =<priority>=: Optional. Priority relative to other pages (0.0 to 1.0).

*/ 3. Plain Text Alternative
If you want something simpler, you can use a =.txt= file containing one absolute URL per line:
#+begin_src text
https://yourdomain.com/
https://yourdomain.com/projects
https://yourdomain.com/blog
#+end_src

*/ 4. Implementation Details
- *Location:* Place the file in your root directory: =https://yourdomain.com/sitemap.xml=.
- *Robots.txt:* Reference the sitemap in your =robots.txt= file so search engines find it:
  #+begin_src text
  Sitemap: https://yourdomain.com/sitemap.xml
  #+end_src
- *Static Site Generators:* If you are using Hugo, Jekyll, or Next.js, they usually generate this file automatically during the build process.
